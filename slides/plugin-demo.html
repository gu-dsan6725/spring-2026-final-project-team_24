What We Built

An Obsidian plugin backed by a FastAPI server that turns your personal notes into an AI-driven study system.

  • Write concepts as markdown notes in your vault
  • The plugin reads your knowledge graph (concepts + edges)
  • An LLM pipeline generates study items tailored to your knowledge
  • You answer, get graded, and progress through difficulty tiers

The Generation Pipeline

A round chains a pre-grading pass + 5-stage loop + per-concept answer grading, all via LLM tool-calling:

Agent Role
Sample Evaluator (pre-round, multimodal) Pre-grades sample reference (md / image) → analysis_notes for Generator
Feasibility Gates whether requested concepts × difficulty is generatable
Generator Produces N items from concepts, edges, examples, images, analysis_notes
Actor ×3 Simulated students solve each item in parallel
Reflector Scores quality, drives refinement (+hardening for hard+)
Round Evaluator Summarizes round quality, supports difficulty progression
Answer Evaluator (per-concept) Verdicts correct / alt_path / misunderstood / untested → Elo mastery

Structured tool-calling (Anthropic Claude Haiku 4.5) — no free-text concatenation.

Iterative Hardening (hard / very_hard / expert)

For difficulty ≥ hard, items go through up to 3 hardening iterations:

  1. Generate items at medium difficulty first
  2. Actor solves them — produces reasoning trajectories
  3. Hardening Reflector analyzes trajectories:
    • “Actor solved trivially → add multi-step reasoning”
    • “High confidence → inject edge cases”
    • “Few concepts used → require cross-concept synthesis”
  4. Reflector also pulls random extra concepts from your graph
  5. Generator re-creates items with hardening directives
  6. Repeat — item(i+1) = f(item(i)), single-slot context (no accumulation)

Five Difficulty Tiers

  • User manually selects difficulty — no forced auto-escalation
  • Hard+ triggers the iterative hardening loop above
  • Extra concepts from your knowledge graph make items harder but solvable

Plugin UI — Context Images & PDF Upload

Inline Context Images

Attach exam paper images as visual context:

  • Attach Image — file picker (png, jpg, webp)
  • From Vault — pick images in your vault

Images are base64-encoded and sent inline with the tool-calling JSON — Claude sees the image alongside concept context when generating items.

PDF → Concept Graph

Upload a PDF paper:

  1. MinerU extracts text to markdown
  2. LLM segments into concept nodes + directed edges
  3. Notes auto-created in vault with YAML frontmatter
  4. Concepts + edges auto-populated in the generator form

Instantly turns any paper into a study-ready knowledge graph.

Plugin UI — Session Flow

Session Resume & Sample Items

Resume

  • Session state lives in the markdown note (YAML frontmatter)
  • Pick any Items/*.md from the dropdown → resume generating rounds
  • Survives Obsidian restarts, backend restarts — fully stateless

Sample Items

  • SampleItems/ folder in your vault
  • Paste exam problems as reference items
  • Plugin reads them and feeds to generator as example_items
  • Generator mimics the style/format of your samples

Answer Grading

The plugin reads your answers from markdown callouts and grades them:

#### Your Answer

> [!quote]- Write your answer here
> P(A|B) = P(B|A) * P(A) / P(B)
> Using the law of total probability for the denominator...

#### Evaluator Feedback  ← injected by plugin
> [!success] Score: 8.5/10
> Strong application of Bayes' theorem.
> Consider explicitly stating the partition used in the denominator.
  • LLM compares your answer against the reference solution
  • Feedback injected directly into the note
  • Scores visible before choosing next round difficulty

Cancel Generation

Misclicks happen — the “Next Round” button has a safety toggle:

  1. Click “Next Round” → button becomes “Cancel Generation”
  2. Click again → generation result is discarded client-side
  3. UI resets — no phantom items appended to your session note

The backend request completes server-side (Obsidian API limitation), but the plugin ignores the response.

Setup for Your Team

1. Clone & install backend

git clone https://github.com/chaowei312/finalproj-backend.git
cd finalproj-backend
pip install -r requirements.txt

2. Configure .env (copy from .env.example)

ANTHROPIC_API_KEY=sk-ant-...
ITEM_GENERATION_PROVIDER=anthropic

3. Start the server

uvicorn app.main:app --reload --port 8000

4. Build & install the plugin

cd plugin
npm install && npm run build

Copy plugin/main.js, plugin/manifest.json, plugin/styles.css into your vault at .obsidian/plugins/knowledge-graph/.

Enable the plugin in Obsidian Settings → Community Plugins.
